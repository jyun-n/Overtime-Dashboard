# 연장 관리 대시보드 — 관리자 매뉴얼

본 문서는 **관리자(ADMIN 권한)** 를 대상으로 합니다.
일반 사용자의 대시보드 기능은 [사용자_매뉴얼.md](./사용자_매뉴얼.md)에 별도로 정리되어 있으며, 본 문서에서는 **관리자 전용 기능** 만 다룹니다.

> 관리자는 사용자의 모든 대시보드 기능을 동일하게 사용할 수 있으며, 추가로 상단 **Admin Console** 내비게이션을 통해 **관리 / 데이터 업로드** 페이지에 접근합니다.

---

## 0. 시작하기

### 0-1. 접속 / 권한

- 관리자 계정으로 로그인하면 상단에 **Admin Console** 헤더가 표시되고, 다음 메뉴를 사용할 수 있습니다.
  - **관리** (`/admin/users`) — 계정 관리
  - **데이터 업로드** (`/admin/uploads`) — 엑셀 업로드 / 다운로드
  - **연장 관리 대시보드** (`/dashboard`) — 사용자와 동일한 분석 화면

### 0-2. 초기 admin 계정

- 시스템 최초 구축 시 **admin 계정은 DB에 직접 INSERT** 됩니다(운영자 작업).
- `admin` 계정은 일반 계정과 달리 **정보 수정·탈퇴 처리가 불가** 하며, 비밀번호 재설정 시 **마스터 비밀번호** 확인이 추가로 요구됩니다.

---

## 1. 계정 관리 (`/admin/users`)

### 1-1. 화면 구성

상단 탭으로 **계정 관리 / 계정 생성** 을 전환합니다.

### 1-2. 계정 관리 탭

전체 계정이 표로 나열됩니다. 컬럼: `순번 / 사번 / 소속부서 / 직급 / 사원명 / 아이디 / 권한 / 관리`.

#### 일반 계정의 관리 버튼
| 버튼 | 동작 |
| --- | --- |
| **수정** | 소속부서, 직급, 권한(USER/ADMIN)을 변경합니다. |
| **비밀번호 재설정** | 해당 계정의 새 비밀번호를 지정합니다. |
| **탈퇴** | 계정을 탈퇴 처리합니다. *(관리자(ADMIN) 권한 계정은 탈퇴 불가)* |

#### `admin` 계정의 관리 버튼
| 버튼 | 동작 |
| --- | --- |
| **비밀번호 재설정** | 마스터 비밀번호 입력 후, 새 비밀번호로 변경합니다. |
| **로그 보기** | 시스템 전체의 로그인 기록·변경 내역을 조회합니다. |

> `admin` 계정은 정보 수정·탈퇴가 차단되어 있습니다. 마스터 비밀번호는 운영자만 보관하세요.

### 1-3. 계정 생성 탭

신규 계정을 등록합니다. 필수 항목:

- **사번** (숫자만 입력)
- **소속부서**
- **직급**
- **사원명**
- **아이디** (중복 불가)
- **비밀번호** (생성 후 사용자에게 안전한 경로로 전달)
- **권한** — 사용자(USER) / 관리자(ADMIN)

#### 비밀번호 정책 (모든 계정 공통)
- **10자 이상**, 최대 128자
- 영문 대/소문자 · 숫자 · 특수문자 중 **3종 이상** 포함
- 동일 문자를 **4회 이상 연속** 사용 불가
- 너무 흔한 비밀번호 차단(서버에서 검증)

오류 발생 시 화면에 다음과 같은 안내가 표시됩니다.
| 코드 | 의미 |
| --- | --- |
| `too_short` | 10자 미만 |
| `too_long` | 128자 초과 |
| `insufficient_complexity` | 3종 미충족 |
| `repeated_chars` | 동일 문자 4회 이상 연속 |
| `common_password` | 흔한 비밀번호 |
| `username_taken` | 아이디 중복 |

### 1-4. 로그 보기 (admin 계정 전용)

`admin` 행의 **로그 보기** 버튼을 누르면 모달이 열리고 두 종류의 기록을 확인할 수 있습니다.

| 탭 | 표시 항목 |
| --- | --- |
| **로그인 기록** | 시각 / 아이디 / 성명 / 소속·직급 / IP — 최근 200건 |
| **변경 내역** | 시각 / 처리자 / 대상 / 변경 내용(CREATE, EDIT_INFO, EDIT_ROLE, RESET_PW, DELETE) / IP |

각 탭에서 **시작일 / 종료일 / 검색어** 필터로 좁힐 수 있습니다.

## 2. 데이터 업로드 (`/admin/uploads`)

### 2-1. 화면 구성

좌우 2개 카드 + 상단 **로그 보기** 버튼으로 구성됩니다.

| 카드 | 용도 |
| --- | --- |
| **인사정보 파일** | 직원 마스터(이름·사번·부서·직급·직군 등) 갱신 |
| **연장근무 파일** | 해당 **년/월** 의 연장근무 데이터 등록 |

### 2-2. 인사정보 파일 업로드

1. **파일 선택** 을 눌러 `.xlsx` / `.xls` 파일을 지정합니다.
2. 파일 선택 즉시 첫 행들이 미리보기로 표시됩니다.
3. **업로드** 버튼 → "기존 데이터가 덮어써집니다. 계속하시겠습니까?" 경고 → **확인 · 업로드** 로 진행합니다.

> 인사정보는 **전체 덮어쓰기** 방식입니다. 기존 마스터를 모두 대체합니다.

### 2-3. 연장근무 파일 업로드

1. **대상 년/월** 을 먼저 선택합니다(기본값: 직전 달, 2025년 이후).
2. `.xlsx` / `.xls` 파일을 선택합니다.
   - 1행이 비어 있는 형식(2행이 헤더)도 자동 인식됩니다.
3. **업로드** → 덮어쓰기 경고 → **확인 · 업로드**.

> 같은 **년/월** 을 다시 업로드하면 해당 월 데이터만 새로 덮어씁니다.

### 2-4. 엑셀 다운로드

연장근무 파일 카드 우측 상단의 **엑셀 다운로드** 버튼을 사용합니다.

- **시작 월 ~ 종료 월** 을 지정하면 해당 기간의 연장수당 데이터를 엑셀로 저장합니다.
- 저장 파일명 규칙: `<사용자명>_<시작YYYYMM>-<종료YYYYMM>.xlsx`
- 다운로드 이력은 자동으로 **다운로드 기록** 에 남습니다.

### 2-5. 업로드 / 다운로드 로그

상단의 **로그 보기** 버튼으로 모달을 열어 두 탭을 확인할 수 있습니다.

| 탭 | 컬럼 |
| --- | --- |
| **업로드 기록** | 시각 / 아이디 / 업로드 내용(HR / OVERTIME) / 대상월 / IP |
| **다운로드 기록** | 시각 / 아이디 / 다운로더 / 기간 / IP |

시작일·종료일·검색어로 필터링 가능합니다.

## 3. 권한 모델 정리

| 역할 | 접근 가능 페이지 | 주요 권한 |
| --- | --- | --- |
| **USER** | `/dashboard` 만 | 대시보드 조회, 종합/상세 리포트 다운로드 |
| **ADMIN** | `/dashboard`, `/admin/users`, `/admin/uploads` | 위 모든 권한 + 계정 CRUD, 데이터 업로드/다운로드, 로그 조회 |

## 4. 운영 체크리스트

### 매월 정기 업무
1. **인사정보 파일** 이 변경되었는지 확인 → 변경분 있으면 업로드.
2. 해당 월의 **연장근무 파일** 업로드.
3. 대시보드 메인에서 **인사정보 파일 / 연장근무 데이터** 의 업데이트 시각이 올바른지 확인.
4. 종합 탭의 KPI가 정상 표기되는지 확인(0 또는 비정상치 점검).

### 비정기 업무
- 신규 입사자 계정 생성, 퇴사자 계정 탈퇴 처리.
- 사용자 비밀번호 분실 시 재설정.
- 분기/연간 정기적으로 **로그인 기록 / 변경 내역** 검토.

## 5. 트러블슈팅

| 증상 | 가능한 원인 / 조치 |
| --- | --- |
| 업로드 후 데이터가 반영되지 않음 | 파일 형식(헤더 위치, 컬럼명) 재확인. 미리보기에서 헤더가 정상 인식되는지 확인. |
| 사용자 로그인 실패가 누적 | 비밀번호 재설정 또는 15분 대기 후 재시도. 동일 IP에서 무작위 시도 시 일시 차단됨. |
| `admin` 계정의 비밀번호 재설정 실패 | **마스터 비밀번호** 가 올바른지 확인. 마스터 비밀번호는 운영자만 보관. |
| 사내망 다른 PC에서 접속 불가 | Windows 방화벽 인바운드 규칙에 **5174 / 4001 TCP** 허용 필요. |
| 외부에서 접속 불가 | 본 시스템은 **사내망 전용** 입니다(0.0.0.0 바인딩, 외부 노출 X). |
